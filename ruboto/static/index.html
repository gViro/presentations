Installation

No purchase necessary to play!

  • Grab the Android SDK (which includes an emulator).
  • Get a pre-made Ruboto APK off of GitHub downloads.
  • (Links at end).

Make an Android Virtual Device

  • Fire up the "android" GUI tool from the SDK.

Install Ruboto-IRB to the AVD.

  • Use the SDK's "adb" debugger tool.
$ adb install ~/Downloads/IRB-0.2-preview1.apk
-bash: adb: command not found

Oh, and make sure the SDK toolset is on your PATH.

$ export PATH=$PATH:$HOME/android-sdk-mac_86/tools/
$ adb install ~/Downloads/IRB-0.2-preview1.apk

Loading Code

No gems (yet)!

  • So how do you work with libraries?

I originally pushed my entire library to the phone...

$ adb push . /sdcard/jruby/rubotoonacid

This works ok...

/sdcard/jruby is in $RUBYLIB.

But a single script is more distributable for now.

$ find lib/rubyonacid -type f -exec cat {} \; \
-exec echo "" \; | grep -v "require '" >> acid.rb;\
cat $HOME/Projects/rubotoonacid/examples/ruboto.rb >> acid.rb

Scott Moyer called this "one script to rule them all". Muhuhuhaha...

May have to tweak declaration order by hand.

$ edit acid.rb

Now there's just one file for users to download.

$ adb push acid.rb /sdcard/jruby/
80 KB/s (23712 bytes in 0.286s)

You can execute from menu...

But "load" commands from IRB auto-load new versions and don't require mousing.

Writing Code

Java

Toast toast = Toast.makeText(

Ruby

import 'android.widget.Toast'
toast = Toast.make_text(
    $activity,
    "Domo arigato!",
    Toast::LENGTH_SHORT
)
toast.show

ruboto.rb conveniences

$activity

  • The current Activity this script is running within.
  • There are other $variables, but they're all passed to callbacks anyway, so don't use 'em.

ruboto.rb Callbacks

  • Each takes a block to be called when its event occurs.
  • start_ruboto_activity: Call this first. Set up other handlers within this block.
  • setup_content: Should return a RubotoView.new(ruboto_activity) to use as container.
  • handle_create {|bundle|}: Called upon Activity creation, right after setup_content block.
  • handle_start {}
  • handle_pause {}
  • handle_resume {}
  • handle_stop {}
  • handle_key {|view, key_code, event|}
  • handle_click {|view|}
  • handle_draw {|view, canvas|}

Fun stuff from the Android API

Intent

  • Switch Activities
  • CRUD content
  • Dial/SMS contacts, do searches...
java_import "android.content.Intent"
java_import "android.net.Uri"
$activity.start_activity (
    Intent.new(
        Intent::ACTION_VIEW,
        Uri.parse("content://contacts/people")
    )
)

Canvas

  • Draw ovals, rectangles, arcs, lines...
  • Work with clips
  • Copy in bitmaps
handle_draw do |view, canvas|
  paint = Paint.new
  paint.color = Color.argb(128, 128, 255, 0)
  paint.stroke_width = 20
  canvas.draw_line(
    0, 0,
    view.get_width, view.get_height,
    paint
  )
end

Sensors

  • Accelerometer
  • Compass
  • Orientation
setup_content do
  sensor_manager = get_system_service(
    Context::SENSOR_SERVICE
  )
  sensor = sensor_manager.get_default_sensor(
    Sensor::TYPE_ACCELEROMETER
  )
  sensor_manager.register_listener(
    self, sensor, SensorManager::SENSOR_DELAY_UI
  )
  @view = TextView.new($myactivity)
end
handle_sensor_changed do |event|
  @view.text = event.values.join("/")
end
